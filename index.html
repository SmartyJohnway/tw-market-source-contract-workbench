07 Netlify Deploy Checklist

  1. 把本資料夾推到 GitHub，或把 zip 拖到 Netlify Deploys。
  2. Netlify 會讀取 netlify.toml：publish 根目錄，functions 在 netlify/functions。
  3. 部署後打開網站，預設 fetch mode 使用 /api/probe，由 redirect rewrite 到 /.netlify/functions/probe。
  4. 若要放寬 host allowlist，可設定環境變數 PROBE_ALLOWED_HOSTS，以逗號分隔；不建議開 PROBE_ALLOW_ANY_URL=true。

08 正式 evidence 建議

Netlify Function probe 比 public CORS proxy 可靠，但仍是 scout。正式納入專案前，請把同一 URL 的 curl/Python probe 結果、HTTP status、content-type、row count、schema snapshot、空資料日行為與日期範圍寫入 source contract。

本頁 history 僅存 localStorage，不寫 DB，不作 production replay input。

Proxy / Evidence Policy

公用 CORS proxy 只用於快速看資料長相。正式 probe 以官方域名 raw URL、curl、Python requests、Codex/Antigravity sandbox、或你部署的 Netlify Function 為準。

  • Netlify Function：受 allowlist 控制，預設建議。
  • Direct fetch：瀏覽器可能 CORS blocked，不代表來源無效。
  • Public proxy：response 經第三方，不具正式證據力。
  • Custom proxy：建議使用自己可控的 localhost 或內部中繼。
